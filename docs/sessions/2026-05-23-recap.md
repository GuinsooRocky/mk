## 二、分发阻断（必修）

### #1 build-dmg.sh 不拷 dylib —— 发布包在他人机器启动即崩
- `build-dmg.sh` 只拷二进制，漏拷 `libsherpa-onnx-c-api.dylib` / `libonnxruntime.1.24.4.dylib`；
  二进制 install_name 是 `@rpath/...`，本机有 Vendor 路径察觉不到，别人机 `dyld: Library not loaded`。
- 修：组装 bundle 时拷两个 dylib 到 `Contents/MacOS/`（对齐 Makefile run/install），`codesign --deep` 一并签。
- 实证：build 后 zip 从 505KB → 8.1M，`MacOS/` 内含 2 dylib，codesign --strict 通过。

### #2 Package.swift 绝对 rpath
- `/Users/lengmo/Desktop/...` 烤进二进制（泄用户名、不可移植）。
- 修：改相对 `@executable_path/../../Vendor/sherpa-onnx/lib`（裸二进制继续能跑），保留 `@executable_path`(.app)。

## 三、安全（#3，仓库公开后尤为重要）

- RemoteInbox 默认 `enabled=true` + 空 token + 绑 0.0.0.0 = 同网段任何人可往聚焦窗口注入键盘事件。
- 修：默认 `enabled=false`；`start()` 空 token 直接拒启（堵无鉴权，保留 Tailscale 跨机用法——配 token 即可）。

## 四、性能（#4/#5）

- `SenseVoiceEngine` 原 `@MainActor` + 同步 `decode`/`ensureLoaded`，松手/启动卡主线程。
- 修：去 @MainActor，改专用串行队列（`com.lengmo.mk.sensevoice`）+ `async` 接口，`recognizer` 只在队列访问；
  调用点 VoiceModule / WEApp 加 `await`，预热 `Task{@MainActor}` 改真后台 `Task{}`。
