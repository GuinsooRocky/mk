## 二、L4 升级：joined-1-token

### 问题
SA 经常把 1 个英文词拆成 N tokens：
- `transformer` → `trans former`
- `Kubernetes` → `k 八 s` / `K8s`
- `nginx` → `engine x`

原 L4 phrase Lev 要求 token 数相等才能 match → 这类全部漏。

### 修法
新增 `asciiSingleTokenByLength: [Int: [String]]`（单 token ASCII canonical 按长度分桶）。

`applyEnglishPhraseFuzzy` 改成两路径并行：
- **路径 a**: bucket[L] tok-wise（原）
- **路径 b**: window join 去空格 vs 1-token canonical Lev（新）

阈值 `max(1, joinedLen/4)`（跟 L2 看齐，防短词 false positive）。

L=2 优先（best-fit 在小窗口；防 L=4/3 抢功）。

### 实测
- "trans former" → "transformer" (d=0, c=1.00) ✅
- "fast api" 不动（fast api 不在 1-token 索引）✅
- 不再误纠 "trans former fast" → "Transformers"（L=2 先）

### 删 28 条手动错音
跑了一轮 audit（`MK --eval-gate` 105 条全测太慢，改用静态规则分类）：
- **可删（算法救得回）**: 28 条 — depsec/clod/Gemne 等单 token 长词
- **必留（算法救不了）**: 148 条
  - 71 条 canon 太短（≤3 字母如 TAM/SAM/ICE）
  - 59 条 token-mismatch（升级后 L4-joined 能救一部分）
  - 14 条 err 中文（跨字符集，需要算法 D）
  - 3 条 err 太短

只保留算法救不了的硬骨头。新词加入只放 canonical 一行。

## 三、算法 D：拼音→英文反查（L6-cn2en）

### 问题
SA zh-CN 模型只输出中文，所以英文词被听成拼音同音的中文：
- "Option" → "又不慎"（pinyin "youbushen"）
- "Shift" → "shifute"（这个 metaphone 抓得到，但中文比如"时夫特"就抓不到）

之前算法 A/B/C 都不跨字符集 → 14 条 KEEP 错音必须手动登记。

### 修法
硬编码 `englishToCnPinyinHints: [English: [中式拼音变体]]`：

```swift
"Option":  ["youbushen", "youbusheng", "yopushen", "youbusen"]
"Command": ["kemande", "kanmaodi"]
"Shift":   ["shifute", "shifu", "shifuti", "xifute"]
"Window":  ["wendou", "wenduo"]
"Cursor":  ["kerso", "kuerso"]
... 17 个键盘/UI 词
```

启动时 `loadAll` 给字典里实际存在的英文 canonical 反向建索引 `cnPinyinToEnglish: [String: String]`。

`correct()` Layer 6：扫输入中文 substring（2-4 字）算拼音 → 反向索引找英文 canonical → 替换。

### 实测
- "我按 又不慎 想用 shifute" → "我按 Option 想用 Shift" ✅
- L6 confidence = 0.5 + L*0.1 - 0.05（2字 0.65 / 3字 0.75 / 4字 0.85）—— 比 L3 同音字稍低（跨字符集易歧义）

### V1 限制
- 仅覆盖硬编码的 17 个词。说 WebSocket / HashMap / Postgres 等没在表里 → 不工作
- 用户可以 `MK --learn` 手动加，或后续做 V2 启发式 English→pinyin 转换

## 四、用户偏好：空格 → 换行（unbounded）

### 上下文
用户：说"空格"想另起一行。

### 第一版踩坑
加 `("空格", "\n")` 到现有 rules。但 PunctuationNormalizer 的 isolated boundary 检查在中文密排（如"了空格下"）下失败 → 不替换。

### 修法
新增 `unboundedRules`：不要边界检查，全文 replace。

```swift
private static let unboundedRules: [(source: String, target: String)] = [
    ("空格", "\n"),
]
```

应用时调 `result.replacingOccurrences(of:with:)` 全文搜索。

副作用：用户真要打"键盘空格键"会被改。但语音输入语境下这种情况极少。

## 七、关键设计原则汇总（5/10-5/11 累积）

1. **chunk 边界 ≠ 用户语义边界** → SA 假句号必 strip
2. **流式期间贴 raw，松手时一次性大刷新**（飞书思路）
3. **WAV 整段重处理是修 chunk 切碎的唯一干净方案**
4. **算法 > 字典** —— 拼音哈希 + Metaphone + L4-joined + 算法 D 协同消灭 whack-a-mole
5. **84% 的手动错音是算法救不了的硬骨头**（token mismatch / 跨字符集 / 极短词）
6. **Gate 默认阈值 0.5 + 调前必跑 eval** —— 防"走下坡路"的工程纪律
7. **用户偏好硬编码可直接改源**（如"空格→换行"），不上配置 UI 因为是个人项目
